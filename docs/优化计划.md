# ideaRound 产品体验升级与大模型深度赋能计划

## 一、 核心定位与目标重构
当前的系统过于强调“流程配置”与“表单化结构”，导致体验割裂、认知门槛高，核心的“多视角观点碰撞”价值被严重削弱。
未来的产品优化核心方向：**从“繁琐的配置工具”转型为“沉浸式、高智能的 AI 智囊团”**。在保证**“用户意图明确（可长文/可单句）”**这个最核心前提下，通过做减法隐藏复杂流程，并**在多个隐性环节深度引入大模型（LLM）能力**，全面接管评估、提炼、主持等工作，直接向用户呈现多智能体思想碰撞的戏剧张力和决策价值。
执行原则补充：
*   **效果与效率优先**：模型成本不作为当前阶段约束，以输出质量、响应速度、用户可感知价值为第一目标。
*   **控制方案复杂度**：优先使用统一技术栈和最少组件，避免为了“完美架构”引入多套并行方案。
*   **先可用再增强**：每个能力先交付可运行的最小闭环，再在后续版本增强精细度。

## 二、 关键优化方向与大模型(LLM)落地场景

### 1. 意图解析优化：多模态输入与“AI 主持人”无感澄清
**目标**：确保意图明确不跑偏，同时消除前置表单的枯燥感。
*   **支持多模态与长文本解析（LLM 赋能）**：
    *   **落地方式**：支持用户输入一句话，或直接上传需求文档、PDF报告、业务截图。
    *   **LLM 应用**：后台调用大模型进行**“隐式意图抽取”**。用户上传资料后，LLM 自动在后台提取 `Core Goal`（核心目标）、`Constraints`（限制条件）、`Pain Points`（痛点），不再强制要求用户手动填表。
*   **化表单为自然对话（引入 AI 主持人 Agent）**：
    *   **落地方式**：取消现有的“意图探针（Probe）”表单页，直接进入圆桌界面。
    *   **LLM 应用**：在圆桌中引入一位特殊角色——**“圆桌主持人 / 需求分析师”**。如果 LLM 发现提取的意图存在模糊地带，由主持人在圆桌开场时以自然语言发起追问（例如：“这很有趣，但在请技术和产品专家发表意见前，我们能先确定一下这个功能的核心用户群吗？”），在沉浸式对话中完成意图的精准补全。

### 2. 降低上手门槛：完善场景模板库与分层配置
**目标**：消除晦涩的专业术语配置，让新用户 10 秒内体验核心价值；同时保留高阶用户的可玩性。
*   **前台极简，配置后台化**：对普通用户隐藏“灵魂配置”、“角色立场”、“意图锚点”等术语。这些复杂的 Prompt 工程与参数配置全部迁移至**管理后台**，由管理员负责调优。
*   **预设经典会议局（场景模板库）**：
    *   *“产品功能杀手局”*（预设：激进产品经理、保守老用户、技术架构师）
    *   *“职业发展拷问局”*（预设：现实主义长辈、理想主义导师、冷酷HR）
    *   *“商业方案路演局”*（预设：挑剔投资人、合规法务、市场营销专家）
    *   **操作流**：用户点击模板 -> 输入/上传一句话或文档 -> 自动跳过所有配置，直接进入圆桌看 AI 们激烈辩论。
*   **保留“高级模式”（Expert Mode）**：
    *   在新建入口处提供一个不显眼的“高级模式”开关。开启后，老用户依然可以访问原有的角色矩阵、灵魂配置、意图探针等全套自定义流程，满足高阶极客的微调需求。

### 3. 增强掌控感：LLM 驱动的智能进度条与超级导演模式
**目标**：解决讨论过程的“失控感”和“不知何时结束”的焦虑。
*   **大模型驱动的“目标达成度”实时评估（进度条）**：
    *   **落地方式**：在圆桌页面顶部增加一个动态进度条（如：`当前共识度 65% | 痛点已解决 2/3`）。
    *   **异步架构设计**：为了不阻塞主对话流，引入消息队列（如 Celery/Redis）。每隔 2-3 轮对话，触发一个**后台裁判 Agent 的异步任务**，向 LLM 发送当前 Transcript 进行评分。前端通过 WebSocket 或轮询获取打分（0-100%）并渲染动画。
*   **超级导演干预按钮（Quick Actions）**：
    *   **落地方式**：提供一组全局干预按钮：🎯“跑题了拉回来”、⚔️“加大对抗烈度”、💡“我有新点子”、🛑“直接总结”。
    *   **LLM 应用**：点击按钮后，不是生硬地修改系统提示词，而是通过“AI 主持人”转化为控场发言。例如点击“加大对抗烈度”，系统触发一条隐藏 Prompt 给反方 Agent：“用户希望看到更尖锐的批评，请立刻针对上一个观点提出致命漏洞。”

### 4. 缓解信息过载：LLM 实时可视化看板
**目标**：解决多个 Agent 长篇大论造成的文本瀑布流阅读障碍。
*   **限制发言冗余**：修改底层 Prompt，严格限制每个 Agent 每次发言字数（例如不超过150字），强制 AI 抛出“一针见血”的核心观点。
*   **大模型驱动的实时“争议/共识”看板（侧边栏）**：
    *   **落地方式**：在群聊旁边的侧边栏，动态展示当前的“思维导图”或“阵营对抗板”。
    *   **异步架构设计**：同样依托异步任务系统，运行一个低延迟的**书记员 Agent**。它订阅对话事件流，周期性增量读取上下文，输出极简的 JSON 结构（`{"共识": ["..."], "争议点": [{"正方":"...", "反方":"..."}]}`），并推送到前端实时更新看板图表。

### 5. 底层逻辑、数据监控与技术架构治理
**目标**：提升系统的稳定性、灵活性，并为未来的数据驱动迭代建立完善的凭证体系。
*   **全链路数据埋点与监控（数据凭证）**：
    *   **落地方式**：在管理后台建立完善的监控看板。对核心交互（如模板点击、长文本上传、导演按钮使用、AI 响应耗时、对话中断率）进行精准埋点。
    *   **日志持久化**：系统不仅要记录用户的显性操作，还要将大模型的中间态（如裁判 Agent 的打分依据、意图抽取的原始 Prompt 和 Response）结构化落库。这些数据将成为未来微调（Fine-tuning）专属小模型或优化 Prompt 的核心资产（凭证）。
*   **解绑刚性规则**：移除代码中强制要求“必须有黑帽和审计官”的校验逻辑，允许用户根据场景自由搭配 2 个或更多的任意角色。
*   **状态管理后端化（去 localStorage 依赖）**：
    *   **演进规划**：当前重度依赖 localStorage 导致跨设备无法同步、容易丢失。未来规划：1) 完善后端的 Session/Workspace 表结构；2) 引入增量保存机制（防抖提交）；3) 最终实现历史版本控制（Time-travel），允许用户回滚到讨论的某个历史节点。
*   **双向同步的智能画布**：解决目前画布功能与讨论脱节的问题。允许用户在画布上直接修改 AI 生成的结构化内容，修改后，触发 LLM 重新理解用户的调整，并同步到后续的讨论上下文中。

## 三、 实施优先级与迭代计划 (Roadmap)

### Phase 1: 扫除认知障碍，实现“一键入局”（短期，1-2个版本）
*   **核心动作**：
    *   上线“场景模板库”（首页一键创建）及配置的后台管理。
    *   新增“高级模式”开关，隔离普通用户与高阶用户的界面。
    *   引入“AI 圆桌主持人”，用自然开场对话替代前置“探针”表单。
*   **效果衡量指标**：
    *   **新用户转化率**（进入首页 -> 成功开启首场圆桌的比例）提升 50% 以上。
    *   **冷启动耗时**（从点击新建到 AI 首次发言的平均时间）从分钟级降至 15 秒内。

### Phase 2: 大模型深度介入与架构重构（中期）
*   **核心动作**：
    *   搭建统一异步任务系统（为裁判/书记员 Agent 铺路，采用单队列+单工作池的简单形态先落地）。
    *   开发后台裁判 Agent，实现页面顶部的“目标达成度”智能进度条。
    *   上线“超级导演”快捷干预按钮。
    *   支持长文档/PDF上传，后台 LLM 自动提取隐式意图卡片。
    *   核心状态数据逐步迁移至后端数据库，减少对本地缓存的依赖。
    *   建立全链路数据埋点体系与 LLM 交互日志系统。
*   **效果衡量指标**：
    *   **用户留存率与活跃度**：用户主动干预次数（点击导演按钮频次）。
    *   **方案可用性**：单次圆桌的平均轮数（期待稳定在有效收敛区间，而非无意义的长尾废话）。
    *   **数据安全**：跨设备登录的会话恢复成功率达 99% 以上。
    *   **系统稳定性**：LLM 接口的 P99 响应耗时与异常失败率监控。

### Phase 3: 可视化重构与双向协同体验飞跃（长期）
*   **核心动作**：
    *   研发并上线由“书记员 Agent”异步驱动的侧边栏实时“争议/共识看板”。
    *   智能画布功能的深度整合（支持用户编辑、大模型理解编辑意图并双向同步）。
    *   全面的移动端/平板端响应式适配。
    *   实现基于后端的圆桌版本控制与快照回滚。
*   **效果衡量指标**：
    *   **移动端占比**：移动端访问与使用时长占比突破 30%。
    *   **画布交互率**：用户在讨论后对生成的画布/方案进行主动编辑和保存的比例。
    *   **NPS (净推荐值)**：产品口碑与自发分享率。
